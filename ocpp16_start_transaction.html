Understanding How EV Charging Sessions Begin

🔍 What is Start Transaction?

▼
🚀

Definition

Start Transaction is like the "key turning in the ignition" of EV charging. It's the official message sent from a charging station to the management system saying "a charging session has officially begun!"

🔐

Authorization Required

Before starting, the user must be authorized - like showing your membership card at a gym. This can be done with RFID cards, mobile apps, credit cards, or other identification methods.

⚡

What Triggers It

• User swipes RFID card or uses mobile app
• Car is properly plugged into the charging connector
• Authorization is successful
• Charging station is ready and available

📋

What It Contains

• Unique transaction ID
• Which connector is being used
• User identification
• Starting meter reading
• Timestamp of session start

🔄 Complete Start Transaction Process

▼

1. User Arrives

User approaches charging station with EV

2. Authorization

Swipes card, uses app, or enters payment

3. Car Connection

Plugs charging cable into vehicle

4. System Check

Station verifies everything is ready

5. Transaction Starts

Official charging session begins

1User Authorization

What happens: The user identifies themselves to the charging station using their preferred method.

🔑 Real-World Example:

Just like using your keycard to enter your office building, or tapping your credit card to buy coffee. The charging station needs to know who you are before it will provide electricity.

Common Authorization Methods:

• RFID Cards: Tap a physical card (most common)

• Mobile Apps: Use smartphone app to start charging

• Credit Cards: Tap or insert payment card

• QR Codes: Scan code with phone camera

• PIN Entry: Enter access code on keypad

2Physical Connection

What happens: The user connects the charging cable from the station to their electric vehicle.

🔌 Real-World Example:

Like plugging your phone charger into your phone, but much bigger! The charging station detects when the cable is properly connected and the car is ready to receive power.

Safety Checks Performed:

• Proper Connection: Cable is securely attached

• Vehicle Ready: Car confirms it can accept charge

• Safety Systems: Ground fault and overcurrent protection active

• Communication: Car and charger can "talk" to each other

3System Verification

What happens: The charging station performs final checks to ensure everything is ready for safe charging.

🔍 Real-World Example:

Like a pilot's pre-flight checklist - the charging station verifies all systems are go before starting to deliver power. This prevents problems and ensures safety.

Verification Checklist:

• Connector Status: Available and functioning

• User Authorization: Valid and confirmed

• Vehicle Communication: Car is responding

• Electrical Safety: All safety systems operational

• Payment Method: Valid and can be charged

4Start Transaction Message

What happens: The charging station sends an official "Start Transaction" message to the management system.

📨 Real-World Example:

Like checking into a hotel - the front desk (charging station) calls the manager (management system) to say "Guest John Smith has officially checked into room 204 and his stay has begun."

📄 Example: Start Transaction JSON Message

{
  "messageType": "StartTransaction",
  "connectorId": 1,
  "idTag": "RFID12345678",
  "meterStart": 0,
  "reservationId": null,
  "timestamp": "2025-08-07T14:30:00.000Z"
}

What this means: "At 2:30 PM on August 7th, 2025, user with RFID card 'RFID12345678' started a charging session on connector 1, with meter reading starting at 0 kWh."

5Transaction Begins

What happens: The management system acknowledges the start and assigns a unique transaction ID. Power begins flowing to the vehicle.

🎫 Real-World Example:

Like getting a parking ticket when you enter a parking garage - the system gives you a unique number (transaction ID) to track your session from start to finish.

What Happens Now:

• Unique ID Assigned: Transaction gets permanent tracking number

• Power Delivery Starts: Electricity begins flowing to car

• Meter Monitoring: Energy consumption tracking begins

• Status Updates: Regular progress reports to management system

• Billing Clock Starts: Customer charges begin accumulating

📄 Example: Start Transaction Response

{
  "messageType": "StartTransactionResponse",
  "idTagInfo": {
    "status": "Accepted"
  },
  "transactionId": 98765
}

What this means: "Management system says: 'Authorization accepted, your charging session is now transaction #98765, you can start charging!'"

❌ What Can Go Wrong?

▼

Authorization Failures

🚫 Common Issues:

• Invalid Card: RFID card not recognized or expired

• Account Problems: User account suspended or no payment method

• Network Issues: Can't reach authorization server

• Card Reader Malfunction: Hardware not working properly

Result: User sees "Authorization Failed" message and cannot start charging.

Connection Problems

⚠️ Physical Issues:

• Loose Connection: Cable not properly inserted

• Incompatible Connector: Wrong type of charging port

• Vehicle Not Ready: Car's charging system has issues

• Cable Damage: Charging cable has electrical problems

Result: Charging cannot start even with valid authorization.

System Errors

🔧 Technical Problems:

• Connector Fault: Charging port has electrical issues

• Communication Error: Can't reach management system

• Power Supply Issue: Not enough electrical capacity available

• Safety System Triggered: Emergency stop activated

Result: Transaction may start but immediately fail or never begin.

📚 Summary

▼

Key Takeaway: Start Transaction is like the official "handshake" between user, charging station, and management system that begins a charging session. It ensures proper authorization, safety checks, and creates a trackable session for billing and monitoring.

✅

What We Learned

• Start Transaction officially begins charging sessions
• Multiple steps ensure safety and proper authorization
• Creates unique transaction ID for tracking
• Can fail at several points for safety reasons

🎯

Real-World Impact

• Enables secure access to charging
• Provides audit trail for billing
• Ensures safety before power delivery
• Allows remote monitoring and support
