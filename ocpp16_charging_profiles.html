🎛️ What are Set Charging Profile Events?

▼
🎯

Definition

Set Charging Profile events are like remote control commands that the management system sends to chargers. Think of them as detailed instructions - like programming your home thermostat or setting timers on your coffee maker.

📅

What They Control

• When to charge: Specific time schedules
• How fast to charge: Power limits during different periods
• Which connector: Specific charging ports
• How long: Duration and validity periods

🚀

Real-World Benefits

• Cost savings: Charge during cheaper electricity hours
• Grid protection: Prevent electrical overloads
• Customer control: Honor specific charging preferences
• Safety compliance: Stay within electrical limits

⚡

Two Main Types

• TxProfile: Rules for specific charging sessions
• TxDefaultProfile: Default rules for all future sessions
• Priority system: Specific rules override defaults
• Flexible scheduling: Daily, weekly, or one-time patterns

📋 TxProfile

▼

Specific Session Rules

What it is: Like giving a charger a personalized schedule for a specific charging session. Similar to setting a custom alarm schedule on your phone for different days.

⏰ When System Sends It:

• When a car plugs in - Specific charging rules for that session
• During peak hours - Limit power when electricity is expensive
• For scheduled charging - Customer wants specific charging times
• Emergency situations - Reduce power consumption immediately
• Load balancing - Distribute power among multiple cars

🎯 Why It Sends It:

• Cost management: Charge during cheaper hours
• Grid protection: Prevent electrical system overload
• Customer preferences: Honor specific schedules
• Energy optimization: Use renewable energy when available
• Safety compliance: Stay within electrical limits

📄 Example: TxProfile JSON Event

Here's what a specific charging session profile looks like:

{
  "connectorId": 1,
  "csChargingProfiles": {
    "chargingProfileId": 12345,
    "transactionId": 67890,
    "stackLevel": 1,
    "chargingProfilePurpose": "TxProfile",
    "chargingProfileKind": "Absolute",
    "recurrencyKind": "Daily",
    "validFrom": "2025-08-07T06:00:00.000Z",
    "validTo": "2025-08-08T06:00:00.000Z",
    "chargingSchedule": {
      "duration": 86400,
      "startSchedule": "2025-08-07T06:00:00.000Z",
      "chargingRateUnit": "A",
      "chargingSchedulePeriod": [
        {
          "startPeriod": 0,
          "limit": 32.0,
          "numberPhases": 3
        },
        {
          "startPeriod": 14400,
          "limit": 16.0,
          "numberPhases": 3
        },
        {
          "startPeriod": 72000,
          "limit": 32.0,
          "numberPhases": 3
        }
      ]
    }
  }
}

What this means: "For charging port 1, from 6 AM to 10 AM charge at full power (32A), from 10 AM to 2 AM next day charge at half power (16A), then from 2 AM to 6 AM charge at full power again. This is for transaction #67890 and repeats daily."

Field Explanations:

connectorId: 1 Applies to charging port #1 (like specifying "bedroom outlet")
chargingProfileId: 12345 Unique ID for this charging rule (like a recipe number)
transactionId: 67890 Links to specific charging session (like a parking ticket number)
stackLevel: 1 High priority (lower numbers = higher priority)
chargingRateUnit: "A" Power limits in Amperes (electrical current)
startPeriod: 0, 14400, 72000 Time periods in seconds (0=6AM, 14400=10AM, 72000=2AM next day)

🔧 TxDefaultProfile

▼

Default Rules for All Sessions

What it is: Like setting default rules for a charging station that apply to ALL future sessions, unless overridden. It's like setting your home thermostat to a default schedule that applies every day.

⏰ When System Sends It:

• During station setup - Configuring new charging station
• Policy updates - Site owner changes rules for all users
• Time-of-use changes - Electricity rate schedules change
• Load management updates - Building electrical limits change
• Maintenance periods - Reduced charging during maintenance

🎯 Why It Sends It:

• Consistent behavior: All sessions follow same basic rules
• Site-wide policies: Building-specific charging limits
• Default cost optimization: Standard time-of-use schedules
• Safety defaults: Conservative power limits as fallback
• Simplified management: One rule for everyone

📄 Example: TxDefaultProfile JSON Event

Here's what a default profile for all charging sessions looks like:

{
  "connectorId": 0,
  "csChargingProfiles": {
    "chargingProfileId": 99999,
    "stackLevel": 5,
    "chargingProfilePurpose": "TxDefaultProfile",
    "chargingProfileKind": "Recurring",
    "recurrencyKind": "Weekly",
    "validFrom": "2025-08-07T00:00:00.000Z",
    "chargingSchedule": {
      "duration": 604800,
      "startSchedule": "2025-08-07T00:00:00.000Z",
      "chargingRateUnit": "kW",
      "chargingSchedulePeriod": [
        {
          "startPeriod": 0,
          "limit": 7.4,
          "numberPhases": 1
        },
        {
          "startPeriod": 32400,
          "limit": 3.7,
          "numberPhases": 1
        },
        {
          "startPeriod": 64800,
          "limit": 7.4,
          "numberPhases": 1
        }
      ]
    }
  }
}

What this means: "For all charging ports, by default: charge at full power (7.4kW) from midnight to 9 AM, reduce to half power (3.7kW) from 9 AM to 6 PM, then back to full power from 6 PM to midnight. Repeat this pattern every week."

Field Explanations:

connectorId: 0 Applies to ALL connectors (0 means "all ports")
stackLevel: 5 Lower priority - can be overridden by specific profiles
chargingProfilePurpose: "TxDefaultProfile" Default profile for all transactions
chargingRateUnit: "kW" Power limits in Kilowatts (power unit)
duration: 604800 Schedule lasts 604,800 seconds (7 days/1 week)
No transactionId Applies to all future transactions, not specific ones

🔄 Key Differences Comparison

▼
Aspect TxProfile TxDefaultProfile
Scope Specific charging session All future charging sessions
Priority High (typically stackLevel 1-2) Low (typically stackLevel 5-9)
Duration Short-term (hours to days) Long-term (weeks to permanent)
Override Capability Cannot be overridden by defaults Can be overridden by specific profiles
Transaction ID Has specific transactionId No transactionId (applies to all)
Connector ID Usually specific connector (1,2,3...) Often 0 (all connectors)
Use Case "This charging session only" "All sessions unless told otherwise"
Everyday Analogy Setting a one-time alarm Setting your default alarm schedule

📊 Possible Units & Values

▼
⚡

chargingRateUnit Options

"A" (Amperes) Electrical current flow - like gallons per minute through a hose. Values: 6A, 16A, 32A, 63A
"kW" (Kilowatts) Electrical power - like horsepower for electrical devices. Values: 3.7kW, 7.4kW, 11kW, 22kW, 50kW
🔌

numberPhases Options

1 Single-phase power (like most home outlets)
3 Three-phase power (industrial/commercial for faster charging)
🕐

Time Values (All in Seconds)

startPeriod Seconds from schedule start (0 = start time, 3600 = 1 hour later)
duration Total seconds for schedule (86400 = 24 hours, 604800 = 1 week)
🎯

stackLevel (Priority)

1-3 High priority (specific session rules)
4-6 Medium priority (site-specific rules)
7-9 Low priority (default/fallback rules)

📚 Summary

▼

Key Takeaway: Charging profiles are like programmable timers that control when and how fast electric vehicles charge. TxProfile gives specific instructions for individual sessions, while TxDefaultProfile sets standard rules for all charging unless specifically overridden.

✅

What We Learned

• Charging profiles control when and how fast EVs charge
• Two main types: specific session vs default rules
• Priority system allows overrides
• Time-based power scheduling saves costs and protects grid

🎯

Think of It Like

• TxProfile: "For this customer today, charge at these specific times"
• TxDefaultProfile: "Unless told otherwise, always follow this standard schedule"
• Like the difference between a one-time alarm and your default alarm schedule
