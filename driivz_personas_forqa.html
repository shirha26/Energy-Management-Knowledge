🎯 Driivz Customer Personas

QA Testing Guide - Understanding Our Users

🎓 Purpose for QA Teams

This guide helps QA engineers understand our customers' perspectives, needs, and workflows. Use these personas to:

  • Design realistic test scenarios
  • Validate user flows from the customer's viewpoint
  • Identify edge cases based on real usage patterns
  • Ensure features meet actual customer needs
👔

Persona 1: Site Manager (Fleet Operator)

▼
Role Manages charging infrastructure for commercial fleet
Company Size 50-500 vehicles
Tech Savviness Moderate
Primary Goal Maximize uptime, minimize operational costs

📋 Key Responsibilities

  • Monitor charger availability and status
  • Schedule maintenance and repairs
  • Manage driver access and permissions
  • Generate usage reports for management

🎯 Testing Focus Areas

OP (Operator Portal):

Charger status monitoring User access management Reporting functionality Maintenance scheduling

🧪 Test Scenarios

Scenario 1: Multiple chargers going offline simultaneously
Scenario 2: Bulk user permission changes
Scenario 3: Report generation during peak usage times
Scenario 4: Emergency maintenance workflows

⚠️ Pain Points to Test

❌ Delayed status updates causing operational issues
❌ Complex user permission systems
❌ Incomplete or inaccurate reporting data
❌ Difficulty identifying problematic chargers
🚗

Persona 2: EV Driver (End User)

▼
Role Employee using company charging infrastructure
Tech Savviness Varies (Low to High)
Primary Goal Charge vehicle quickly and easily
Device Primarily mobile (DP - Driver Portal)

📋 Key Activities

  • Find available charging stations
  • Start/stop charging sessions
  • Monitor charging progress
  • Review charging history and costs

🎯 Testing Focus Areas

DP (Driver Portal):

Station availability Session start/stop flows Real-time charging status Payment visibility

🧪 Test Scenarios

Scenario 1: Poor network connectivity during session start
Scenario 2: Cable disconnection scenarios (StopTransactionOnEVSideDisconnect)
Scenario 3: Reservation conflicts
Scenario 4: Multiple authentication methods (RFID, mobile app)

⚠️ Pain Points to Test

❌ Unclear error messages
❌ Slow app response times
❌ Confusing charging status indicators
❌ Issues with session termination

🔄 Cross-Persona Testing Scenarios

Multi-User Workflows

1. Session Handoff
Driver starts session → Site Manager monitors → Technician diagnoses issue
2. Authorization Chain
CPO configures access → Site Manager assigns users → Driver authenticates
3. Maintenance Flow
System detects fault → Technician receives alert → Site Manager updated

📡 OCPP Protocol Testing

Based on OCPP 1.6 specification, test these critical flows across personas:

Authorization (Section 4.1)
  • Local vs. remote authorization
  • Offline behavior with Authorization Cache
  • Parent idTag scenarios
Transaction Management (Sections 4.8, 4.10)
  • StartTransaction with various scenarios
  • StopTransaction reasons
  • Transaction data reporting
Smart Charging (Section 3.12)
  • ChargingProfile management
  • Load balancing scenarios
  • Profile conflicts and resolution
Status Management (Section 4.9)
  • All ChargePointStatus transitions
  • Error condition handling
  • Connector-specific vs. Charge Point status

✅ QA Testing Checklist by Persona

For Each Persona, Verify:

Functional Requirements

  • All primary workflows complete successfully
  • Error handling matches user expectations
  • UI/UX appropriate for tech savviness level
  • Device compatibility (mobile/desktop)

OCPP Compliance

  • Relevant OCPP messages handled correctly
  • Status updates reflect actual charge point state
  • Authorization flows work as specified
  • Transaction data accurate and complete

Performance

  • Response times acceptable for user's needs
  • System handles expected concurrent users
  • Real-time updates occur within SLA

Edge Cases

  • Network interruption scenarios
  • Concurrent user actions
  • System recovery procedures
  • Data consistency across personas

📝 Documentation for QA

When testing, document:

  1. Which persona you're testing as
  2. Their goals in the workflow
  3. Expected behavior from their perspective
  4. Pain points discovered
  5. OCPP messages involved (if applicable)

💡 Remember: This helps ensure we're not just testing features, but validating real user experiences.

🎯 Testing Best Practices

Always approach testing from the user's perspective

Validate OCPP compliance for all charging operations
